# 콘텐츠 요청 템플릿

> 새 스레드 글이 필요할 때 이 양식으로 요청하면 `content-playbook.md` 공식에 맞춰 일관되게 뽑을 수 있다.
> 채우기 귀찮으면 "이 주제로 3개 뽑아줘" 한 줄이어도 됨 — 나머지는 공식 기본값으로 채운다.

```
[주제]        예: 속눈썹 재료 고르기 / 리터치 안내법 / 창업 대출 / 첫 손님 받는 법
[카테고리]    정보 · 공감 · 반전 · 전환 중 택1 (기본: 정보)
[후킹 유형]   숫자 · 의외성 · 질문 · 손실회피 · 폭로 중 택1 (기본: 자동)
[개수]        예: 3개
[내 경험/수치] 있으면 적기 (예: "나는 공유샵 6개월 후 1인샵 갔음") → 진짜 경험이 반응을 높임
[CTA 목표]    댓글 · 저장 · 팔로우 · 문의 중 택1 (기본: 댓글)
```

## 예시 요청
> "[주제] 속눈썹 재료 초보가 사면 후회하는 것 / [카테고리] 반전 / [개수] 3개 / [CTA] 저장"

## 산출물 형식 (항상 이 형태로 받음)
- **[본문]**: 후킹 → 공감 → 정보 3~5개 → 반전/CTA
- **[댓글]**: 결정적 정보 or 추가 인사이트 (체류시간·전환용)

발행 후에는 조회수/댓글/저장 수를 함께 기록해서, 어떤 후킹·주제가 먹혔는지 누적 학습한다.
